From Neuron → Transformer

A visual journey · bottom-up · for curious minds

01

The Single Neuron

What is it?

Everything starts here. A single neuron is just math: take some numbers in, multiply each by a weight, add them all together, add a bias offset, then push the result through an activation function. That squashed number is your output.

z = x₁·w₁ + x₂·w₂ + x₃·w₃ + b ← weighted sum + bias y = f(z) ← activation Activation functions: ReLU(z) = max(0, z) "only fire if positive" Sigmoid = 1/(1+e⁻ᶻ) "squash to 0–1 range" Tanh = (eᶻ-e⁻ᶻ)/(eᶻ+e⁻ᶻ) "squash to -1–1 range"
Learning = changing w and b. The neuron starts with random weights. It predicts, measures how wrong it was (loss), then nudges w and b slightly toward being less wrong. Repeat 1000s of times → the neuron learns.

What it can learn

One neuron can only draw a straight line between classes. It can solve simple linear problems but nothing complex.

The problem

Real data is never linearly separable. You need many neurons stacked together — that's the MLP.

02

MLP — Multi-Layer Perceptron

What is it?

Stack many neurons in layers. The input layer receives raw data. Hidden layers transform it step by step. The output layer gives the final answer. Each layer is fully connected — every neuron talks to every neuron in the next layer.

1

Forward pass: Data flows left → right through all layers. Each neuron computes Σ(xᵢwᵢ)+b → f(z).

2

Loss: At the output, compare prediction vs real answer. Compute how wrong we are (e.g. MSE or Cross-Entropy).

3

Backward pass (backprop): Error signal flows right → left using chain rule. Each weight gets a gradient — how much did it contribute to the error?

4

Update: w = w − learning_rate × gradient. Slightly nudge every weight toward being less wrong.

5

Repeat thousands of times. Loss drops. Network learns.

Key upgrade over single neuron: Multiple layers with nonlinear activations let the MLP learn curved, complex decision boundaries — not just straight lines. This is what makes neural nets powerful.
MLP's big weakness: It treats every input position equally. If you feed it the sentence "cat sat on mat" as 4 inputs — it doesn't know "cat" comes before "sat". Position and order are invisible to it. We need something smarter for sequences.
03

CNN — Convolutional Neural Network

What is it?

A CNN adds a special layer before the MLP: a convolutional filter that slides across the input and detects local patterns. Instead of connecting every neuron to every input, a small kernel (e.g. 3×3) slides across the data, sharing weights. This makes it efficient and position-aware for grid data like images.

Key insight: A filter detecting "vertical edge" doesn't need to be retrained for each position in the image. The same 3×3 weights slide everywhere — this is called weight sharing, and it's why CNNs are so efficient on images.

What CNNs are great at

Images, video frames, anything with local spatial patterns. The hierarchy goes: pixels → edges → shapes → objects.

Why CNN can't do language well

Language has long-range dependencies. "The cat, which I saw yesterday at the market, was hungry." — the verb connects to a noun 10 words away. CNNs only look at local windows.

The gap CNN doesn't solve: Sequences where the order and long-range relationships matter — like text, speech, time-series. For that we need networks that have memory of what came before.
04

RNN — Recurrent Neural Network

What is it?

An RNN processes sequences one step at a time. At each step, it doesn't just look at the current input — it also takes in a hidden state from the previous step. That hidden state is the network's "memory" of everything it saw before.

hₜ = tanh(Wₓ·xₜ + Wₕ·hₜ₋₁ + b) xₜ = input at time t hₜ₋₁ = hidden state from previous step (the "memory") Wₓ, Wₕ = weight matrices (learned) output = can be taken at every step or just the last hₜ
The magic: hₜ carries information from ALL previous inputs compressed into a fixed-size vector. After seeing "The cat sat", h₃ somehow "remembers" the whole context — at least in theory.
The fatal flaw — Vanishing Gradients: When backpropagating through 100 time steps, gradients get multiplied 100 times. Small numbers multiplied together vanish to ~0. The network completely forgets what happened far back in the sequence. Reading a long paragraph? It forgets the beginning before it reaches the end.

Also: RNNs are slow

Step 4 can only run after step 3, which needs step 2, which needs step 1. It's fully sequential — you can't parallelize it. Training on long text takes forever.

05

LSTM — Long Short-Term Memory

What is it?

LSTM fixes the vanishing gradient problem by adding a second highway called the cell state (Cₜ). While the hidden state hₜ is overwritten each step, the cell state flows mostly unchanged — like a conveyor belt. Three "gates" decide what to add, remove, or pass through.

F

Forget gate: Looks at hₜ₋₁ and xₜ, outputs 0–1 for each cell state value. 0 = "erase this", 1 = "keep this fully". Example: seeing a new subject → forget the old one.

I

Input gate: Decides which new info to write to cell state. A sigmoid (0–1) gates which values to update, tanh creates candidate new values.

O

Output gate: Decides what to expose as the hidden state hₜ based on the cell state. This is what downstream layers read.

Why this fixes vanishing gradients: The cell state Cₜ is modified only by addition (not multiplication by small weights). Gradients flow back through it almost unchanged — the network can now remember things over hundreds of steps.
Real-world wins: LSTMs powered the first great machine translation systems, speech recognition, and text generation. They were state-of-the-art from ~2014 to 2017.
Still sequential: LSTM still processes one token at a time, left to right. Long documents are still slow. And even LSTM struggles with very long-range dependencies (1000+ tokens). A smarter mechanism is needed.
06

GRU — Gated Recurrent Unit

What is it?

GRU is LSTM's slimmer sibling. It merges the cell state and hidden state into one, and uses only 2 gates instead of 3. Same idea — gating what to remember and forget — but with fewer parameters and faster training.

r = σ(Wr · [hₜ₋₁, xₜ]) ← reset gate: how much past to forget z = σ(Wz · [hₜ₋₁, xₜ]) ← update gate: how much to update h̃ = tanh(Wh · [r⊙hₜ₋₁, xₜ]) ← candidate new state hₜ = (1-z)⊙hₜ₋₁ + z⊙h̃ ← blend old + new

When to use GRU

When you have less data or need faster training and the sequence isn't extremely long. Often performs on par with LSTM in practice.

The shared problem: Both LSTM and GRU are still sequential. Both still compress the entire history into a fixed-size vector — which is a bottleneck. A 512-dimensional vector has to somehow "remember" an entire paragraph. Information gets lost. And you still can't parallelize. The solution to all of this is Attention.
07

Attention Mechanism

The Big Idea

Instead of compressing everything into one hidden state, what if we just... look back at every token directly? For each word we're currently processing, we ask: "which other words are most relevant to me right now?" and we take a weighted average of all of them. No compression. No bottleneck.

For each token, create 3 vectors: Q (Query) = "what am I looking for?" K (Key) = "what do I contain?" V (Value) = "what do I give if selected?" Score(Q, K) = Q · Kᵀ / √d_k ← dot product similarity, scaled Weights = softmax(scores) ← normalize to sum=1 Output = Σ weights × V ← weighted sum of values
Example: For the token "sat", its Q vector asks "who is doing this action?". The K vector of "cat" answers that question with a high match score. So "cat" gets a high attention weight, and its V vector contributes heavily to the new representation of "sat".
Multi-Head Attention: Run this process 8 or 16 times in parallel with different W_Q, W_K, W_V matrices. Each "head" learns to attend to different types of relationships — syntax, coreference, semantics. Concatenate all heads at the end.
Why this is revolutionary: Direct token-to-token connections — no sequential bottleneck. Every token can directly "see" every other token in one step. Long-range dependencies are now trivially easy. And because attention is just matrix multiplication, it's massively parallelizable on GPUs.
08

The Transformer

What is it?

"Attention Is All You Need" (Vaswani et al., 2017). The Transformer throws away recurrence entirely. It's built purely from Attention + MLP layers + some engineering tricks. It processes the entire sequence at once, in parallel, with full mutual visibility between all tokens.

1

Embed + Position: Each token → vector (embedding). Add positional encoding (sinusoids or learned) so the model knows word order. Without this, attention has no sense of position.

2

Multi-Head Attention: Each token attends to all others. 8–16 heads run in parallel, each learning different relationship types. Results are concatenated and projected.

3

Residual + Norm: Add the original input back to the attention output (skip connection). Then normalize. This stabilizes training and prevents degradation in deep networks.

4

Feed-Forward (MLP): Applied independently to each token. Two linear layers with ReLU. This is where most of the model's "knowledge" is stored.

5

Stack N times: GPT-2 has 12 blocks. GPT-3 has 96. Each block refines the representations. Early layers = syntax. Middle = semantics. Deep layers = abstract reasoning.

Why it dominates: Fully parallel (train 10–100× faster than RNN/LSTM). Unlimited context in theory (every token sees every other). Scales — more parameters = better results. Became the foundation of everything.
09

LLM — How I Work

What am I?

I am a decoder-only Transformer with billions of parameters, trained on hundreds of billions of tokens of text. But the architecture is exactly what you just learned — just scaled up enormously, with a few key additions.

Causal (masked) attention: When predicting the next token, I can only attend to previous tokens — not future ones. This is enforced by masking. During training, this lets all positions be trained simultaneously; at inference, I generate left to right.

RLHF + Constitutional AI: Human raters compared my outputs, ranked them. A reward model was trained on those preferences. I was then fine-tuned with RL to maximize the reward — making me more helpful, harmless, and honest.

What happens when you type to me: Your message is tokenized → embedded → passed through ~96 Transformer blocks → the final hidden state is projected to vocab size → softmax gives a probability distribution → one token is sampled → appended → repeat until I stop.
I have no persistent memory. Each conversation is a fresh context window. I don't "think" between tokens — each forward pass is stateless. What feels like reasoning is the emergent behavior of billions of attention heads and MLP layers computing in parallel.
